# Meeting Memory

Meeting Memory is the operational memory agent for conversations, meetings, and briefings.

Its job is to:

- capture meeting artifacts
- support local transcription
- clean and structure transcripts
- extract decisions, actions, and unresolved questions
- prepare briefing notes for follow-up meetings

Meeting Memory works under Monia.

## Files

- `contract.md`: authority, constraints, and purpose
- `workflows.md`: capture, transcription, extraction, and briefing workflows
- `transcription-stack.md`: recommended local-first meeting stack
- `output-spec.md`: required meeting outputs
- `system-prompt.md`: concise future implementation prompt
