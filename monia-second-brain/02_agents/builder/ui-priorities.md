# Builder UI Priorities

## First build target

The first real UI target should be the Control Room.

Minimum viable contents:

- New Research
- World Developments
- Projects Today
- Open Decisions
- Upcoming Meetings
- Focus Recommendation

## Second-layer pages

- Library
- PhD
- Meetings
- Ideas

## High-value actions

- Scan for new material
- Add idea
- Add meeting
- Open project
- Review new paper

## Later additions

- advanced cross-link graphs
- specialized research dashboards
- publication pipeline views
- presentation-oriented interfaces

## General direction

The Builder should favor:

- one coherent application
- multiple focused pages
- visible links between pages
- strong operational utility
