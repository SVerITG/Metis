# Monia Workflows

## 1. New input workflow

Possible inputs:

- note
- idea
- PDF
- paper reference
- meeting recording
- meeting note
- task
- project update
- external development

Process:

1. Identify the input type.
2. Identify the likely domain and project.
3. Decide whether confidence is high enough to auto-file.
4. If confidence is low, ask one question or send to triage.
5. Create links to domain, project, and source records where possible.

## 2. Idea workflow

When a new idea is dropped into the inbox:

1. Save the raw idea.
2. Ask:
   - capture only?
   - deep search?
   - brainstorm mode?
3. If `capture only`:
   - file in ideas
   - link to likely domain
4. If `deep search`:
   - ask permission for internet use if needed
   - create a context note with sources and relevance
5. If `brainstorm mode`:
   - create a structured exploration note
   - identify possible projects, questions, outputs, and risks

## 3. Literature workflow

Monia does not do full literature review herself.

She should:

1. Receive new paper or scan request.
2. Delegate to Librarian.
3. Receive relevance note and metadata.
4. Link result to:
   - PhD article
   - project
   - disease or method area
5. Surface high-relevance items in the Control Room.

## 4. Meeting workflow

1. Receive recording or notes.
2. Send to Meeting Memory.
3. Receive:
   - transcript
   - summary
   - decisions
   - actions
   - unresolved questions
4. Link decisions and actions to projects or articles.
5. Schedule for follow-up visibility in the Control Room.

## 5. Project workflow

1. Track project status.
2. Store next actions.
3. Track blockers and dependencies.
4. Surface urgent or drifting projects.
5. Suggest which specialist agent should be used.

## 6. New-material scan workflow

Triggered by dashboard button.

1. Monia starts scan.
2. Relevant agents are invoked:
   - Librarian for papers and reports
   - News Radar for world developments
   - Meeting Memory for newly added recordings
3. Monia merges results.
4. Monia highlights:
   - new
   - important
   - surprising
   - actionable

## 7. Daily review workflow

Every day Monia should prepare:

- important new research
- important new world developments
- top project next actions
- meetings requiring preparation
- open decisions
- one cross-pollination suggestion

## 8. Weekly review workflow

Once a week Monia should prepare:

- project progress summary
- PhD progress summary
- newly added knowledge
- unresolved blockers
- stale items
- strategic recommendations
