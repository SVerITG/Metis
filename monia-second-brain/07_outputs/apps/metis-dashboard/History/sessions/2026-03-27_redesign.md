## Decisions Made

- Decision: Use visNetwork for mind maps and cluster maps
  Rationale: Native Shiny integration, supports click events (used for click-to-filter), node sizing, edge styling, browser-rendered physics layout. See ADR-003.
  Alternatives considered: networkD3 (less Shiny integration), igraph (server-side only), ggraph (static).

- Decision: Warm earth-tone palette
  Rationale: Reduces contrast fatigue for a personal daily-use tool. Cream/teal/rust palette reads better over long sessions than clinical dark-blue defaults. See ADR-004.
  Alternatives considered: Dark mode (rejected), default shinydashboard blue (rejected).

- Decision: Graceful fallback for visNetwork in all modules
  Rationale: visNetwork is a new dependency. If it is absent (e.g. on a fresh install before check_setup.R runs), the app must not crash — it should show a helpful message instead.
  Alternatives considered: Hard dependency with error (rejected — bad UX).
