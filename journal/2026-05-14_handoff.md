## Known issue — run.sh powershell.exe call

`run.sh` now contains:
```bash
_win_key=$(powershell.exe -NoProfile -NonInteractive -Command \
    "[Environment]::GetEnvironmentVariable('ANTHROPIC_API_KEY','User')" 2>/dev/null | tr -d '\r\n')
[ -n "$_win_key" ] && export ANTHROPIC_API_KEY="$_win_key"
```

**Risk:** On some WSL setups, calling `powershell.exe` in a `$()` subshell hangs and blocks the server from starting. If Stan reports the dashboard won't open, remove this block entirely and instead add to `~/.bashrc`:
```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```
This is the simpler and more reliable approach.
