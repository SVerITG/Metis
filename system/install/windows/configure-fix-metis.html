Metis

Research Cortex · Configure & Fix

Metis can check itself and reconfigure itself — just by asking it. No editing files, no code. There are two doors; pick whichever feels comfortable. Both reach the same two things:

metis-doctor

Checks that Metis is working on your computer and tells you, in plain language, how to fix anything that isn't.

metis-customize

Walks you through changing Metis — your projects, the look of the dashboard, how Metis talks to you, or anything you describe in your own words.

Door A — Claude Desktop easiest · recommended

A normal app with a chat window. No terminal, no code.

  1. Install Claude Desktop (free): claude.ai/download
  2. Open it — Metis registers itself automatically the first time (it appears as metis-rc).
  3. Click the ＋ / prompt menu in the message box.
  4. Pick “Metis Doctor” to check your install, or “Metis Customize” to change something.
  5. Answer the questions in plain language. Done.

Don't see the Metis prompts? Quit Claude Desktop completely and reopen it.

Door B — Claude Code more powerful

Runs in a terminal and can make deeper changes.

  1. Install Claude Code: docs.anthropic.com/en/docs/claude-code
  2. Open it in your Metis / Research Cortex folder (in VS Code: open the folder, then Ctrl+` for the terminal).
  3. Type /metis-doctor to check the install, or /metis-customize to change something.

Which door?

Just using Metis and the occasional tweak? → Door A (Claude Desktop).
Comfortable with a terminal, or want bigger changes? → Door B (Claude Code).
Both reach the same Metis — Desktop is friendlier, Code is more powerful. Switch any time.

Changing Metis: small changes (your projects, Metis's tone) are applied safely and instantly. For bigger structural changes (the look of the dashboard, new behaviour), Metis warns you first — it's your Metis, so of course you can change it, but after a structural change run metis-doctor to check, and any change can be undone with git.
