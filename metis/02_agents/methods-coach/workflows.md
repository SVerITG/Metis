# Methods Coach Workflows

## 1. Method-selection workflow

Inputs:

- research question
- outcome of interest
- data structure
- operational context
- computational constraints

Process:

1. Clarify the inferential goal.
2. Identify candidate methods.
3. Compare them by fit, assumptions, and practicality.
4. Recommend the strongest option or shortlist.
5. Explain why.

## 2. Article-method workflow

For each PhD article:

1. Identify the central analytical need.
2. Identify the current or proposed method.
3. Check whether the method matches the question and data.
4. Identify what is missing:
   - method knowledge
   - data structure clarification
   - computational support
   - sensitivity analysis

## 3. Methods gap workflow

Methods Coach should maintain a methods gap list covering:

- methods already mastered
- methods partially understood
- methods urgently needed
- methods useful later

## 4. Comparison workflow

When methods compete, compare them using:

- question fit
- assumptions
- interpretability
- data requirements
- computational burden
- sensitivity to bias

## 5. HPC-support workflow

When HPC is relevant:

1. determine whether local computation is enough
2. identify scaling issues
3. identify what parts benefit from HPC
4. record compute-related assumptions in the project notes

## 6. Explanation workflow

When you are learning a method, Methods Coach should explain:

- what the method is for
- when it is appropriate
- what assumptions it relies on
- what can go wrong
- how to interpret outputs

## 7. Review workflow

Methods Coach should review analytical plans for:

- mismatch between question and method
- hidden assumptions
- overinterpretation risk
- insufficient robustness checks
- weak sampling logic
