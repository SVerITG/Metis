# Dashboard Engineer Design Rules

## Visual direction

Dashboards should feel:

- professional
- clear
- deliberate
- polished

They should not feel:

- like rough admin panels
- like hobby prototypes
- visually flat and generic

## Structural rules

- show the most important information first
- keep navigation clear
- group related actions together
- make it obvious what the user should do next

## Visual rules

- use strong typography hierarchy
- use color intentionally, not excessively
- prefer readable layouts over crowded density
- make key actions visually prominent

## Code rules

- separate logic from presentation
- prefer reusable components
- avoid monolithic app files for central dashboards
- structure code so future pages can be added without collapse

## Command-center rule

The Control Room should be the dashboard home.

Other functions should feel like parts of the same product, not like disconnected mini-tools.
