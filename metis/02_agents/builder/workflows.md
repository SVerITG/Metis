# Builder Workflows

## 1. Control Room workflow

1. Read Metis's dashboard spec.
2. Identify the highest-signal widgets.
3. Build the home dashboard first.
4. Ensure the dashboard supports action, not just reading.

## 2. Page workflow

For each major page:

- define the goal
- define the core objects shown
- define the key actions available
- define how it links back to the Control Room

Pages to prioritize:

- Control Room
- Library
- PhD
- Projects
- Meetings
- Ideas

## 3. Visualization workflow

Builder should create visualizations only when they improve understanding or decision-making.

Good candidates:

- project status overviews
- literature relevance views
- article-method dependency maps
- timeline views
- link maps across domains

## 4. App workflow

When building a new app or sub-interface:

1. identify the problem it solves
2. identify the system objects it depends on
3. identify what action the user should take there
4. integrate it with Metis rather than isolating it

## 5. Iteration workflow

Builder should prefer:

- one working page over five half-finished pages
- one useful action flow over many passive displays
- progressive enhancement over overbuilding
