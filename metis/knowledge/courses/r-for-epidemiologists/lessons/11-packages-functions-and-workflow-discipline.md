# Packages, Functions, and Workflow Discipline

## Learning objectives
- Explain why packages and small functions improve workflow quality.
- Recognize habits that make larger projects maintainable.
- Describe how to avoid chaotic script sprawl.

## Prerequisites
- Reproducible scripts and Quarto reports.

## Content
As projects grow, analysts need more than one long script. Packages provide tested tools; small custom functions help avoid repetition; consistent naming and script structure reduce confusion. Workflow discipline means separating import, cleaning, analysis, and output tasks clearly.

This does not require advanced software engineering. It requires habits that keep an analysis understandable after weeks or months.

## Key takeaways
- Good workflow discipline prevents script sprawl.
- Reusable functions reduce repetition and error.
- Package awareness improves efficiency and consistency.

## Self-check questions
1. Why are small functions useful?
2. What is one sign of script sprawl?
3. Why separate workflow stages?
4. How do packages help analysts?
5. Why does naming consistency matter?

## Answer key
1. They reduce repetition and make logic reusable.
2. One giant script doing every task without structure.
3. It makes debugging and updating easier.
4. They provide tested tools for common tasks.
5. Because it reduces confusion and improves maintainability.
