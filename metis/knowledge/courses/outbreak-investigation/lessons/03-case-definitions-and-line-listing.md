# Case Definitions and Line Listing

## Learning objectives
- Build practical suspect, probable, and confirmed case definitions.
- Explain what a line list is and how it supports outbreak analysis.
- Recognize the trade-off between sensitivity and specificity in case definitions.

## Prerequisites
- Prepare, verify, and define the problem.

## Content

### Section 1: Case definitions are operational tools
A case definition is a standardized set of criteria used to decide whether a person is counted in the investigation. It is not the same thing as a clinical diagnosis. In outbreak work, the goal is consistent counting and triage.

### Section 2: Suspect, probable, confirmed
Suspect definitions are usually broad and sensitive. Probable definitions add contextual or laboratory support. Confirmed definitions rely on definitive laboratory or other high-specificity evidence. Different tiers are useful because response often starts before confirmation is complete.

### Section 3: The line list
A line list is the working dataset of the outbreak. Each row is a case and each column captures information such as identifier, age, sex, residence, onset date, exposure, laboratory status, outcome, and location. Line lists should be simple enough to maintain during response and structured enough to analyze quickly.

### Practical example
In a cholera investigation, the initial suspect definition may prioritize sensitivity to find cases quickly. As laboratory data accumulate, the team may tighten the probable and confirmed definitions while still retaining the broader suspect group for operational tracking.

## Key takeaways
- Case definitions exist to standardize counting and action.
- Sensitivity and specificity need to match the stage of the investigation.
- A clean line list is one of the most valuable outbreak tools.

## Self-check questions
1. Why is a case definition not identical to a clinical diagnosis?
2. Why might suspect definitions be intentionally broad?
3. What is the main function of a line list?
4. Name four useful variables for a line list.
5. Why should teams document changes to the case definition?

## Answer key
1. Because it is a surveillance and investigation tool designed for consistency, not individualized care.
2. To maximize early case detection when uncertainty is high.
3. To organize case data for monitoring, description, and analysis.
4. Onset date, residence, age, laboratory status, exposure, or outcome.
5. Because changes affect comparability of counts over time.
