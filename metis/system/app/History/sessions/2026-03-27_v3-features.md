# Session Log — 2026-03-27 (v3.0 feature build)

## Summary

Eight features were designed and implemented in a single session, advancing the Metis Dashboard from a polished read-only tool (v2.0) to a fully interactive daily work surface (v3.0). The session covered new intelligence features (morning brief, spaced repetition, full-text search), planning features (kanban board, PhD milestone timeline), and exploration features (global knowledge graph, article gallery). Two new module files were created. Five existing modules were completely rewritten. The SQLite schema was extended with two new tables. The helpers and data_store layers each received six new functions.

## Decisions Made

- Decision: Search and Graph placed under nav_menu("More") dropdown
  Rationale: Primary navbar already at 7 tabs; 9 tabs would overflow on standard laptop screens. Search and Graph are on-demand features, not daily-loop tabs.
  Alternatives considered: Direct tab addition (overflow), combined Search+Graph tab (confusing UX).
  ADR: ADR-006

- Decision: Single shared observeEvent for all kanban card status transitions
  Rationale: Per-card observers registered inside renderUI loops fire multiple times after UI re-renders (duplicate observer problem). Single observer on a shared Shiny.setInputValue input is registered once at startup.
  Alternatives considered: Per-card observeEvent (duplicate firing), shinyjs::onclick (same problem).
  ADR: ADR-007

- Decision: SR reveal is client-side CSS toggle only
  Rationale: Answer HTML is already in the page; showing it is a presentation concern. No server round-trip needed until the user rates the card.
  Alternatives considered: shinyjs::toggle (dependency + latency), conditionalPanel (server state update required).
  ADR: ADR-008

## Issues Encountered

- Issue: Kanban observer duplication when registering per-card observers inside renderUI
  Resolution: Solved by moving all status transition logic to a single observeEvent on input$kanban_move, populated via Shiny.setInputValue from JavaScript buttons on each card.

- Issue: Navbar overflow risk at 9 tabs
  Resolution: Grouped Search and Graph under nav_menu("More") dropdown.
